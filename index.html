Git Commands

git init: for initialising

git status: for checking the status

git add "." or file name: for adding files to repository whereas "." represents all files or we can mention file name

git commit -m "message": for commiting

git log --oneline: for checking the logs of commit(after "--" no space is required)

git log: for detailed commit details

git checkout "id" "file name": will give the file with that commit

git reset HEAD "file name": will revert the staging of added files

git checkout -- "file name": will restore back to original file (after "--" space is required)

git reset "fiel name": will unstage a staged file, but leave working directory unchanged

git reset: reset the staging file area to the last commit without disturbing the working directory

git remote add origin "url for new respository": Add the remote online repository

git push -u origin master: push the local git repository to the origin to master branch

git clone "url of repository": for cloning the repository available online to local system

NPM(node package module)

npm -v: for checking version

node -v: for checking node version

npm init: for initialising

npm install "package-name" --save-dev: where dev represents for saving this in packet.json file
